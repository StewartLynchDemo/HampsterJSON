Step 1: Creating the GitHub Repository

  1. Once you have a GitHub Account, you can create a new repository

  1. When the dialog opens, you can

    1. Enter the Repository name, like Hampster.JSON

    2. Give the repository a description (Optional)

    3. Unless you have a paid GitHub account, the repository must be public.

    4. Click on Add README and it will generate a ReadMe file automatically for you

    5. Click on Create repository

Adding WebPublishing via Pages

  1. Once the repository is created, click on Settings

  1. At the next page, in the left column click on Pages

  1. On the next pages, makes sure that

    1. Deploy from a branch is selected

    2. Select the main branch and

    3. Save

  1. Once you have saved, it will take a few minutes for GitHub to set up the Pages, but eventually, if you keep refreshing the page, you will

    1. get a link to your new website

    2. A button will be there to take you to the website

  1. If you visit the site, you will see the ReadMe file that is just the description of the repository

  1. Make note of the link as this is going to be the root* url for your JSON files. In ,my case the url is https://stewartlynchdemo.github.io/HampsterJSON

Uploading your JSON Files

What you want to publish is your JSON files.

  1. Back in the GitHub repository, go back to the Code tab

  2. Tap on the Add file button and choose Upload Files

  1. On the next window, you can drag your two JSON Files into the area specified

  1. Once you do that,

    1. they will be uploaded to repository and

    2. you need to click on Commit changes

  1. In the end, you will see them in the repository listing

Updating your JSON Files

To update your JSON files, you repeat the same process as above for adding the files in the first place. So long as the files have the same name, they will overwrite the previous version

 

Viewing your published JSON files.

You can view your published JSON files in a browser. If you can do this, then we can code Xcode to import the JSON and decode it and load it in to your SQL Tables.

Note: you upload the files, it takes 2 or 3 minutes for the Pages component of GitHub to process the files or updates

  1. To view your published JSON, you can use your root url followed by the name of the JSON file. So in my case the two urls are: https://stewartlynchdemo.github.io/HampsterJSON/contacts.json and https://stewartlynchdemo.github.io/HampsterJSON/stations.json
